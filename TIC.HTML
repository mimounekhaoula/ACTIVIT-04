CHAPITRE 03 :


Le modèle OSI:
1/ les composants matériel d’un réseau informatique :
un réseau informatique est composé de différents éléments matériels, parmi lesquels se trouvent : les cartes réseau, les switches et le HUB.
• les cartes réseau : sont des composants installés dans les ordinateurs pour leur permettre de se connecter à un réseau, qu'il s'agisse d'un réseau local ou d'Internet. Elles sont connectées à une prise Ethernet ou Wi-Fi et jouent un rôle crucial dans la communication entre les ordinateurs sur le réseau.
• les switches : sont des composants matériels qui permettent de connecter plusieurs appareils (ordinateurs, imprimantes, serveurs) sur un réseau local. Ils utilisent des adresses MAC pour permettre la communication entre ces différents appareils et stockent les informations dans une table de correspondance.
• HUB : est un composant matériel qui permet de connecter plusieurs appareils sur un réseau local. Contrairement à un switch, le hub transme les données d’une machine a l’ensenble des machines connectees a cet hub.
• 2/ Le modèle OSI :
Le modèle OSI (Open Systems Interconnection) est un modèle de référence pour les communications de réseau qui définit sept couches distinctes, chacune ayant une fonction spécifique liée à la communication de données entre les appareils de réseau. Ce modèle est largement utilisé comme référence pour la conception et la mise en œuvre de réseaux informatiques.
Les sept couches du modèle OSI sont les suivantes :
• La couche physique : est responsable de la transmission des données sur le support physique. Elle détermine les caractéristiques électriques, mécaniques et fonctionnelles des interfaces de communication physique, permettant ainsi aux appareils de communiquer entre eux sur un réseau.
• La couche liaison de données : est responsable de la transmission des données entre les appareils qui partagent un même support physique. Cette couche divise les données en trames et ajoute des informations de contrôle d'erreur pour garantir que les données sont transmises sans erreur. Elle permet également de réguler le flux de données entre les appareils pour éviter une saturation du réseau.
• La couche réseau : est responsable du routage des données à travers le réseau.
• La couche transport : est responsable de l'acheminement des données de bout en bout entre les appareils.
• La couche session : est responsable de l'établissement et de la gestion des sessions entre les appareils.
• La couche présentation : est responsable de la conversion des données dans un format qui peut être compris par l'appareil de destination.
• La couche application : permet également la gestion des sessions, l'authentification et la sécurité des communications.
Le modèle OSI est une norme théorique de communication réseau en sept couches, qui n'est plus largement utilisée dans les protocoles de communication de réseau modernes tels que TCP/IP. Cependant, il est encore utilisé comme une référence importante pour comprendre les principes de base de la communication de réseau et peut être utile pour la conception et la résolution de problèmes de réseau.

Le modele TCP/IP :
Le modèle TCP/IP est un modèle de référence pour les communications de réseau, utilisé pour la conception et la mise en œuvre des réseaux informatiques modernes. Il divise les communications en quatre couches principales, contrairement au modèle OSI qui en divise sept :
1. Couche Application : fusionne les 3 couches du modele OSI (Couche Application ,Couche Présentation, couche session ), responsable de la communication entre les applications sur les appareils de réseau.
2. Couche Transport : cette couche est responsable de l'acheminement des données de bout en bout entre les appareils.
3. Couche Internet : cette couche est responsable du routage des données à travers le réseau.
4. Couche Accès réseau : cette couche est responsable de la transmission des données sur le support physique.
L’adressage :
Les communications dans un réseau obéissent à des règles :
• Adressage qui permet d'identifier de manière unique les deux unités en communication
• Architecture qui définit les rôles endossés par les deux unités
• Protocoles qui assurent l'échange des données
adressage IP :
Une adresse IP (Internet Protocol) est le numéro qui identifie chaque ordinateur dans les réseaux qu’il appartient. Il y’a deux mode d’adressage IPV4 et IPV6 :
1. ADRESSE IPV4:
Chaque paquet contient adresse source sur 32 bits ou 4 octets tel que chaque octet represente en decimale separe par un point.
2. ADRESSE IPV6 :
Les adresses sont codées sur 128 bits decoupees en 8 mots de 16 bits en hexadecimale séparés par le caractère «:».
L’adresse IPV4
Les 4 octets composent deux parties une partie des bits de valeur supérieur pour addresse réseau et l’autre pour l’adresse des hôtes, tel que la partie hôtes détermine le nombre des machines possible sur ce réseau.
Les différents types d’adresse IPV4: l’adressage IPV4 s’effectué selon 4 types :
1) L’adresse réseau : tous les bits de la partie hôtes=0, tel que tous les hôtes(machines) de ce réseau ont la même partie réseau.
2) L’adresse de diffusion : il s’agit d’une adresse dans laquelle tous les bits de partie hôtes=1, permette à un pc d’envoyer des paquets à l’adresse de diffusion de réseau.
3) L’adresse hôtes : On attribue les valeurs situés entre l’adresse réseau et l’adresse de diffusion sur {machines} situé sur cette réseau.
4) Le masque de réseau et de sous réseaux : Tous les bits de la partie réseaux = 1.

Internet et Web
1) Introduction
Le texte explique que le World Wide Web (WWW) est un système hypertexte public sur Internet qui permet de consulter des pages accessibles sur des sites. Le Web est une des applications d'Internet et a été inventé par Tim Berners-Lee et Robert Cailliau plusieurs années après Internet. Le principe du World Wide Web est d'accéder à des documents textuels situés sur des machines accessibles par Internet et reliés entre eux par un mécanisme de lien hypertexte. Les liens hypertexte permettent d'ouvrir des pages en suivant une URL (Uniform Resource Locator), qui est le nom unique de la page et contient l'adresse du serveur et l'adresse de la page sur le serveur.
2) Aspects Techniques du Web
Le texte explique que les trois mécanismes de base du Web sont les suivants :
1. L'URL qui permet d'accéder à un ensemble de ressources et qui peut faire appel au protocole DNS pour localiser les ressources.
2.HTTP qui est un protocole de niveau applicatif basé sur le paradigme client-serveur et qui est sans état.
3. HTML qui est un langage de description de pages Web et qui permet d'inclure du texte, des images et d'autres objets. Il permet également de créer des liens hypermédias entre les pages et de faire de la programmation déclarative.
3) La Page Web et le Site Web:
La page web est un document multimédia qui décrit élément par élément avec un langage de description HTML dans un fichier html , interprèté et affiché par un navigateur, elle est localisée sur internet à l'aide d'une adresse URL , elle permet d'accéder à d'autres pages en suivant des liens.
Le site web est un regroupement de plusieurs pages web autour d'une thématique commune , emanant d'une même entité à partir d'une page d'accueil commune et à partir d'un URL localisé généralement sur le même serveur, il est structuré sous forme de dossier et fichiers.

CHAPITRE 05

Système de Gestion de Version
1. Introduction:
Dans ce chapitre, on définit Git et GitHub, et on comprend pourquoi Git est si populaire et pourquoi les systèmes de gestion de version sont indispensables aujourd'hui. Qu’est-ce que Git ?
Git est un logiciel de gestion de version créé par Linus Torvalds qui permet de conserver un historique des modifications d'un projet, facilitant ainsi la gestion de projets collaboratifs et est devenu un outil indispensable pour les développeurs.
2. A quoi sert concrètement un système de gestion de version:
Git permet une gestion efficace de projets collaboratifs en conservant un historique des changements effectués sur des fichiers. Avec un serveur distant contenant l'historique des modifications effectuées, les développeurs peuvent synchroniser leur dossier local avec le serveur et coexister différentes versions d'un même fichier. Cela permet une organisation efficace et un accès facile à l'historique des modifications pour l'ensemble du projet..
3. modèle centralisé vs modèle décentralisé:
Les logiciels de gestion de version se basent sur deux modèles : le modèle centralisé et le modèle décentralisé (ou distribué). Le modèle centralisé centralise le code source du projet sur un serveur distant, tandis que le modèle décentralisé permet à chaque utilisateur de télécharger et d'héberger le code source sur sa propre machine. Git a popularisé le modèle distribué, qui offre des avantages tels que la simplicité et la flexibilité du travail hors ligne, ainsi qu'une meilleure sécurité car chaque utilisateur possède une copie complète du projet qui peut servir de sauvegarde en cas de corruption du serveur central.
4. Qu’est-ce que GitHub:
Git est un outil de gestion de version qui permet de sauvegarder les modifications effectuées sur un projet et de revenir à une version antérieure. Un dépôt, ou "repository" en anglais, est l'ensemble des fichiers et de leurs versions stockés sur un serveur central. GitHub est le plus grand hébergeur de dépôts Git, où une grande partie des dépôts sont publics et accessibles à tous pour contribuer à leur développement en proposant de nouvelles fonctionnalités. Git est un logiciel, alors que GitHub est un service en ligne d'hébergement de dépôts Git qui sert de serveur central pour ces dépôts.

7. Paramétrage de Git:
Après l'installation de Git, nous allons paramétrer le logiciel en renseignant un nom d'utilisateur et une adresse e-mail que Git utilisera par la suite. Nous utiliserons la commande git config pour cela, qui permet de modifier les variables de configuration de Git. Nous entrerons ensuite les commandes suivantes: git config --global user.name "Votre Nom" et git config -- global user.email "Votre Adresse Email" en remplaçant les informations par votre propre nom et votre adresse e-mail.

Vérifiez l'enregistrement de vos informations en tapant "git config user.name" et "git config user.email".
8. Démarrer un dépôt Git:
Le dépôt Git est la copie d'un projet et il peut être créé soit en important un répertoire existant, soit en clonant un dépôt Git existant. Nous allons découvrir comment faire cela plus tard. Avant cela, il est important de comprendre le fonctionnement général de Git et comment il gère les informations.
9. La gestion des informations selon Git:
Git gère les données sous forme d'instantanés ou "snapshots", ce qui signifie qu'à chaque validation ou enregistrement, Git prend un instantané du contenu de l'espace de travail et enregistre une référence à cet instantané dans une base de données locale. Cela permet à la plupart des opérations de Git d'être réalisées localement sans avoir besoin d'être connecté à un serveur central, ce qui rend les opérations plus rapides.
10. Les états des fichiers:
Git utilise un système de suivi des modifications des fichiers, classés en "suivi" s'ils appartiennent au dernier instantané capturé par Git ou en "non suivi" s'ils ne sont pas indexés. Les fichiers suivis peuvent être "modifiés", "indexés" ou "validés", tandis que les nouveaux fichiers sont non suivis jusqu'à ce qu'ils soient indexés. Lorsqu'un instantané est pris, les fichiers faisant partie de celui-ci sont considérés comme "validés" et le cycle recommence, ce qui permet de suivre efficacement les modifications d'un projet avec Git.
11. Les zones de travail:
Le fonctionnement de Git se base sur trois zones de travail : le répertoire de travail pour travailler sur les fichiers extraits de la base de données de Git, la zone d'index pour stocker les informations sur les fichiers qui feront partie du prochain commit, et le répertoire Git pour stocker les méta-données et la base de données des objets du projet. Les modifications sont travaillées dans le répertoire de travail, indexées dans la zone d'index, puis validées pour être ajoutées au répertoire Git.

12. Créer un dépôt Git à partir d’un répertoire existant:
Pour importer des projets dans Git, on utilise la commande Git clone pour copier les fichiers localement ou depuis un serveur distant. On peut initialiser un dépôt Git vide avec la commande git init, qui créera un sous-répertoire .git. Pour indexer des fichiers, on utilise git add et pour valider les fichiers en base, on utilise git commit en ajoutant un message pour documenter les modifications. Il est important de bien documenter chaque commit pour faciliter la compréhension des modifications. Après le commit, la commande git status renvoie un message indiquant que tous les fichiers sont sous suivi de version et enregistrés en base, sans aucune modification depuis le dernier commit. Enfin, git push permet de pousser les modifications locales vers le dépôt distant pour que les autres membres de l'équipe puissent y accéder.

La deuxième façon de démarrer un dépôt Git est de cloner localement un dépôt Git déjà existant. Pour cela, on va utiliser la commande Git clone. Créez un repo sur GitHub:

14. Ajouter ou modifier des fichiers dans Git:
On travaille sur un projet avec des fichiers et un dépôt Git pour gérer les versions. On utilise git add et git commit pour enregistrer les modifications et ajouts de fichiers dans le dépôt. Git log affiche l'historique des commits dans l'ordre du plus récent au plus ancien, avec la somme de contrôle SHA-1, le nom et l'e-mail de l'auteur, la date et le message du commit.
16. Annuler des modifications apportées à un fichier :
La gestion de version permet de revenir à un état antérieur d'un projet enregistré. Après un commit, si des modifications ne donnent pas les résultats attendus, on peut utiliser les commandes "git checkout -- nom-du-fichier" ou "git restore" pour revenir à l'état précédent des fichiers.
17. Qu’est ce qu’une branche ?:
la création d'une branche dans Git permet de développer et tester de nouvelles fonctionnalités sans impacter le projet de base. Contrairement à d'autres systèmes de contrôle de version, Git utilise une approche légère pour les branches en créant simplement un pointeur vers un commit existant. Chaque commit contient des informations telles que l'auteur, le message et un pointeur vers l'instantané du contenu indexé ainsi que des pointeurs vers les commits précédents. La branche par défaut dans Git est "master" et se déplace automatiquement à chaque nouveau commit. Toutefois, il est possible de la renommer. En somme, créer une nouvelle branche dans Git est facile et léger car cela ne nécessite pas de recopier l'intégralité du répertoire de travail, mais simplement de créer un nouveau pointeur.
